Force-Update Hardening — превью изменений

Статический мок-ап того, как теперь выглядит админ-страница «Обновление приложения» и мобильные окна обновления. Это НЕ живое приложение — просто визуальный обзор пунктов 1–3, чтобы ревьюить без деплоя.

1. Админ-страница «Обновление приложения» П1П3

Три порога на платформу: min (жёсткий форс), recommended (мягкое окно), последний build в сторе (защитный потолок). Поля с ошибкой подсвечиваются, кнопка «Сохранить» блокируется.

Минимальный build — ниже него блокирующее окно (0 = не форсить). Рекомендуемый build — мягкое закрываемое окно. Поле «Последний build в сторе» не даёт случайно заблокировать пользователей на неопубликованную версию.

📱 iOS

Ниже этого build показывается закрываемое окно «Позже / Обновить» (раз в день). 0 = не показывать.

Build, реально опубликованный в сторе. Защищает от форса на ещё не вышедшую версию. 0 = не указано.

📱 Android ошибки валидации

⚠️Превышает последний build в сторе (15). Сохранение заблокировано — пользователи остались бы без доступного обновления.
⚠️Превышает последний build в сторе (15) — нельзя рекомендовать версию, которой ещё нет.

Ниже этого build показывается закрываемое окно «Позже / Обновить» (раз в день). 0 = не показывать.

Текущее: iOS ≥ 14, Android ≥ 13
Бэкенд тоже валидирует (422) — даже если обойти UI, сервер отклонит min/recommended выше «последнего build в сторе». Двойная защита.

2. Confirm-модалка перед сохранением П1

Резюме того, что именно произойдёт и для кого — нельзя сохранить «вслепую».

⚠️

Подтвердите изменение обновления

iOS: заблокирует всех с build < 14 (жёсткий форс на 14). iOS: мягкое окно «Позже/Обновить» для build от 14 до 15. iOS: последний build в сторе — 16. Android: жёсткий форс ВЫКЛЮЧЕН (0). Android: последний build в сторе — 15. Убедитесь, что указанные билды реально доступны в сторах. Изменение применяется при следующем запуске приложения у пользователей.

3. История изменений + откат П2

Каждое сохранение пишется в audit-лог (кто/когда/что). «Откатить» загружает старые значения в форму — дальше обычное «Сохранить» (та же валидация + новая запись аудита).
